# Data Model

GlipMath uses one private Google Sheet with three worksheets:

- `question_bank`
- `whitelist`
- `answers`

The app also provides matching CSV templates under `data/templates/`.

## `question_bank`

Required columns:

| Column | Type | Notes |
| --- | --- | --- |
| `id_question` | int | Unique question ID |
| `source` | str | Source or origin label |
| `statement` | str | Question text |
| `choice_a` | str | Required |
| `choice_b` | str | Required |
| `choice_c` | str | Required |
| `choice_d` | str | Required |
| `correct_choice` | str | One of `A/B/C/D/E` |

Optional columns:

| Column | Type | Notes |
| --- | --- | --- |
| `choice_e` | str | Optional 5th alternative |
| `is_active` | bool-like | Defaults to `true` when blank |
| `topic` | str | Optional taxonomy |
| `difficulty` | str | Optional level |
| `explanation` | str | Optional feedback text |

Rules:

- Active questions must have valid `id_question`, `source`, and `statement`.
- Active questions must populate `choice_a` through `choice_d`.
- `choice_e` is optional.
- `correct_choice` must reference a populated choice.
- Inactive rows are ignored by the app.
- Duplicate `id_question` values are treated as blocking validation errors.
- Malformed active rows are skipped by the app and should be fixed manually.

## `whitelist`

Required columns:

| Column | Type | Notes |
| --- | --- | --- |
| `id_user` | int | Unique internal user ID |
| `email` | str | Unique email used for authorization |

Optional columns:

| Column | Type | Notes |
| --- | --- | --- |
| `name` | str | Preferred display name |
| `is_active` | bool-like | Defaults to `true` when blank |

Rules:

- Authorization is based on normalized email matching.
- Email normalization means lowercase + trimmed whitespace.
- If `name` is blank, the app falls back to the Google profile name or the email.
- Inactive users are denied even if they authenticate successfully.
- Duplicate normalized emails are blocking validation errors.

## `answers`

Required columns:

| Column | Type | Notes |
| --- | --- | --- |
| `id_answer` | int | Unique answer event ID generated by the app |
| `id_user` | int | Whitelist user ID |
| `email` | str | Normalized email |
| `id_question` | int | Question ID |
| `selected_choice` | str | Chosen alternative |
| `correct_choice` | str | Correct alternative |
| `is_correct` | bool | Evaluation result |
| `answered_at_utc` | ISO timestamp | UTC event time |
| `answered_at_local` | ISO timestamp | Local event time in configured timezone |
| `time_spent_seconds` | float | Elapsed time from question render to submit |
| `session_id` | str | Streamlit session identifier |

Optional columns:

| Column | Type | Notes |
| --- | --- | --- |
| `source` | str | Copied from the question row |
| `topic` | str | Copied from the question row when available |
| `app_version` | str | App version string |

Rules:

- `answers` is append-only.
- Existing answer rows are never updated.
- Every submission creates exactly one new row.
- Empty `answers` history is valid for a new deployment.

## Derived Metrics

### Day streak

Definition used by this repository:

- Count consecutive local calendar days with at least one answer.
- The streak must end on today or yesterday in the configured timezone.
- If the latest activity is older than yesterday, the streak is `0`.

### Question streak

Definition used by this repository:

- Sort a user's answers in reverse chronological order.
- Count consecutive correct answers from the newest answer backward.
- Stop at the first incorrect answer.

### Leaderboard

Definition used by this repository:

- Population: all active whitelisted users
- Score: total correct answers
- Tie-breakers:
  1. total answers descending
  2. email ascending

Displayed position example:

- `#3 / 27`

## Question Delivery

This MVP uses a simple deterministic rule:

1. serve unseen active valid questions first
2. once all are seen, choose randomly among all active valid questions

The currently displayed question is pinned in Streamlit session state until the student answers or requests the next question.
